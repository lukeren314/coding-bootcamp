Welcome to the completely free Python coding boot camp!

Introduction

 Do you want to be able to automatically find all upper div courses in your department sorted by average GPA? In this course, you'll learn the fundamentals of Python and how to apply it to real-world applications. If your goal is to take the ICS 31 credit exam, switch into CS or a related field, or simply build your practical skills, this course is for you! This boot camp will be split up into 10 weeks of content, with 3 parts per week, like a 4-unit college course.

Course Content

 This course will be taught mainly using interactive lectures and practice problems. We will be using Google Colab (installation is easy), which allows you to run code in the browser and also edit it in real-time. The first half of this course will cover the content in ICS 31 at UCI, the fundamentals of Python, which is what you will need if you want to pass the ICS 31 credit exam. The second half of this course will cover some advanced Python concepts and real-world applications for Python. If you already know Python, feel free to skip to the second half of this course. One of the main projects of this course is to build is a search program that will look through all of the courses at UCI, filter them out based on your preferences (i.e. department, unit count, etc.), and then display them sorted by average GPA. I haven't written all of the modules yet, which means this list will be updated as the quarter goes on.

Course Guidelines

 Make sure that you follow **ALL** of these guidelines if possible:
  • This boot camp is FAST and HARD. Do not feel discouraged if you fall behind or struggle to solve the practice problems. I designed the practice problems to be A. types of problems I encounter all the time and B. very non-trivial.
  • Schedule a specific time to take each module part every week (i.e. Monday, Wednesday, Friday at 2pm). Set a calendar reminder even if you don't use one. This boot camp is as much work as a 4-unit class (if not a little more) so treat it like one.
  • Do not skip the Coding Activities and Practice Problems! These are where you'll really learn the content.
  • Do not look at the solutions before you at try the problem. Seriously. You won't learn anything if you do. But DO look at them once you come up with an answer. If your answer is not similar to the solutions, try and figure out why, and see how you can improve yours.
